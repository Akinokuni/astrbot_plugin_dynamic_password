# astrbot_plugin_dynamic_password

**动态入群密码** 是一个为 AstrBot 设计的插件，旨在通过半天（12小时）时间步长的动态密码机制，解决 QQ 群静态入群密码易被泄露和倒卖的问题，同时相比传统 TOTP 方案降低用户门槛。

## ✨ 核心特性

- **长效动态密码**：每 12 小时（00:00 和 12:00）自动更新一次密码，有效阻断外部死链。
- **自动审核**：接管入群申请，自动匹配申请理由中的密码，验证通过即自动批准。
- **人性化容错**：提供时段切换后的 15 分钟容错窗口，避免用户在临界点因密码失效被拒。
- **多群独立配置**：支持为不同群组设置独立的密钥（Secret）和规则。

## 🛠️ 安装与配置

### 1. 安装插件

将本插件文件夹放入 AstrBot 的 `plugins` 目录中，或通过 AstrBot 插件管理指令安装。

### 2. 初始化配置

插件安装后，需要管理员在群内通过指令进行配置才能生效。

#### 指令列表

| 指令 | 权限 | 描述 | 示例 |
| --- | --- | --- | --- |
| `/动态密码配置 <群号> enable true` | 管理员 | 开启插件功能 | `/动态密码配置 123456 enable true` |
| `/动态密码配置 <群号> secret <值>` | 管理员 | 设置安全种子（必填） | `/动态密码配置 123456 secret my_secret_key` |
| `/动态密码配置 <群号> length <值>` | 管理员 | 设置密码长度（默认6） | `/动态密码配置 123456 length 6` |
| `/动态密码配置 <群号> reject true` | 管理员 | 验证失败自动拒绝（默认否） | `/动态密码配置 123456 reject true` |
| `/动态密码配置 <群号> msg <值>` | 管理员 | 设置拒绝文案（默认：动态密码错误） | `/动态密码配置 123456 msg 密码错误，请获取最新密码` |
| `/加群密码 [群号]` | 所有人 | 获取当前有效密码 | `/加群密码 123456` |

**推荐配置步骤：**
1.  设置安全种子：`/动态密码配置 <群号> secret <随机字符串>`
2.  开启功能：`/动态密码配置 <群号> enable true`
3.  （可选）设置验证失败自动拒绝：`/动态密码配置 <群号> reject true`

## 🧮 算法逻辑

本插件采用自定义的轻量级哈希算法生成动态密码，确保计算快速且具有抗猜解性。

### 1. 计算因子
算法依赖以下三个因子：
1.  **日期 (Date)**: 格式为 `YYYYMMDD`（如 `20260308`），基于 **北京时间 (UTC+8)**。
2.  **时间槽 (Slot)**: 
    - `AM`: 00:00 - 11:59
    - `PM`: 12:00 - 23:59
3.  **密钥 (Secret)**: 管理员配置的字符串，作为盐值（Salt）。

### 2. 生成公式
```python
raw_str = f"{date_str}-{slot}-{secret}"
hash_hex = sha256(raw_str).hexdigest()
password = extract_digits(hash_hex)[:length]
```

**示例：**
- 日期：20260308
- 时间：下午 (PM)
- 密钥：my_secret
- 原始字符串：`20260308-PM-my_secret`
- SHA256 哈希：`a1b2c3d4...`
- 提取数字并截取前6位：`123456`

### 3. 容错机制
考虑到用户获取密码和提交申请之间可能存在时间差，插件设计了 **15分钟容错窗口**。
- 在每天的 `00:00 - 00:15` 和 `12:00 - 12:15` 期间。
- 系统会同时认可 **当前时段** 和 **上一时段** 的密码。
- 例如：在 `12:05` 提交申请，如果填写的是上午的密码，依然可以通过验证。

## ⚠️ 注意事项

- 请妥善保管 `Secret`，不要在公开场合泄露。
- 建议定期更换 `Secret` 以提高安全性。
- 插件依赖 AstrBot 的入群申请事件监听，请确保 Bot 具有管理员权限。
